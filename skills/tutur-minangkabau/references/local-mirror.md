# Local Mirror

## Source Mirror

- Language/register: Bahasa Minangkabau.
- Region/context: Sumatra Barat, rantau Minangkabau, and Minangkabau communities elsewhere.
- Broad family context: Austronesian / Malayo-Polynesian.
- Glottolog: Minangkabau, glottocode `mina1268`.
- ISO 639-3: `min`.
- Badan Bahasa peta bahasa material covers language mapping for Sumatra Barat.
- Kemendikdasmen repository includes `Kamus Minangkabau-Indonesia`.
- Public grammar references include `Tata Bahasa Minangkabau`.

## Local Usability

Use this file as the offline substitute for source orientation. It is not a verbatim dictionary mirror.

The skill can produce everyday Minangkabau-flavored wording, review Indonesian interference, and create short bilingual public text when the vocabulary is covered by local notes or user samples. For adat formulas, ceremonial speech, legal/community titles, or dialect-specific public use, mark the relevant parts for speaker review.

## Vocabulary And Semantics

- Minangkabau is not just "bahasa Padang". Avoid collapsing all Minangkabau voices into urban Padang speech.
- Adat, kinship, matrilineal relations, surau, nagari, and rantau terms can carry social meaning. Do not freely translate them without context.
- Rantau contexts may prefer a more mixed Indonesian-Minangkabau style than village/community contexts.
- Keep honorifics, clan/suku names, place names, and institutional names unchanged unless the user provides the local form.
- Avoid forcing Indonesian official terms into local forms without a verified source.

## Generation Rules

1. Ask for audience when the text involves elders, adat, nagari, formal invitations, or ceremony.
2. Build the meaning in Indonesian first.
3. Use Minangkabau terms only when supported by local references or user samples.
4. For public copy, a bilingual Indonesian-Minangkabau shape is often safer.
5. Keep uncertain adat and kinship terms marked for review.

## Starter Lexicon Policy

Do not bulk-copy the dictionary. Add short verified terms only when their meaning and usage are clear. Prefer semantic notes and phrase patterns that help an agent avoid fake fluency.
