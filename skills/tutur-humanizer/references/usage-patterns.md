# Usage Patterns

## Ordinary Output

For a plain rewrite request, return the rewritten Indonesian text directly. Add notes only when the user asks for review, when meaning changed, or when a register choice matters.

## Sample Tasks

- Humanize AI-sounding Indonesian.
- Convert stiff formal writing into clear professional Indonesian.
- Make casual copy sound natural without forcing slang.
- Clean official text while preserving EYD/KBBI-aligned diction.
- Rewrite sales copy so the benefit is concrete.
- Review whether a paragraph sounds translated from English.

## Register And Semantics

- Default: natural baku-santai, readable for public use without sounding bureaucratic.
- Resmi: complete sentences, standard punctuation, no slang, no inflated claims.
- Profesional: direct business Indonesian, no corporate padding.
- Gaul/santai: keep rhythm short, use slang lightly, avoid over-trying.
- Sales: lead with pain point or benefit, then proof or action.
- Academic/technical: keep terms precise and avoid unnecessary simplification.

## Review Checklist

- Are facts, names, numbers, dates, and claims preserved?
- Does the text still sound Indonesian rather than English in Indonesian words?
- Are repeated AI phrases removed?
- Is the register consistent with the audience?
- Are slang and regional words intentional?
- Does the rewrite add unsupported claims or proof?

## Output Patterns

- Rewrite only: output text.
- Review: issues first, then improved version.
- Multi-register: label each variant clearly.
- File edit: edit the file, then summarize material changes.
