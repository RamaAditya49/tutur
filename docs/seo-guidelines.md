# SEO Guidelines For Tutur Skills

Tutur skills are published as local skill folders and ClawHub entries. The main SEO surface for each skill is `SKILL.md`, not a separate per-skill `README.md`.

## Required SEO Surface

Every public skill should include:

- a clear skill name in the H1,
- a frontmatter `description` that contains the searched language/register name,
- a `Search And Discovery` section near the top of `SKILL.md`,
- local examples in `references/examples.md`,
- source notes in `references/sources.md`,
- install/discovery coverage from the root `README.md` and `docs/language-skills.md`.

## Search Terms

Use honest, user-searchable phrases:

- `Bahasa <Name>`
- `kamus Bahasa <Name>`
- `contoh Bahasa <Name>`
- `belajar Bahasa <Name>`
- `terjemahan Indonesia <Name>`
- `translate Bahasa <Name>`
- `AI skill Bahasa <Name>`
- local dialect/register names, for example `Sunda lemes`, `Jawa krama`, `Banjar Hulu`, `Sasak Lombok`

Do not promise certified translation, official legal wording, ritual authority, or native-speaker correctness unless the skill has expert-reviewed material.

## Content Shape

Good SEO text for this repo has three parts:

1. searchable language/register phrases,
2. concrete use cases,
3. safety boundary.

Example:

```md
## Search And Discovery

Relevant searches: Bahasa Sunda, kamus Bahasa Sunda, contoh Bahasa Sunda, belajar Bahasa Sunda, terjemahan Indonesia Sunda, translate Bahasa Sunda, Sunda lemes, Sunda loma, undak-usuk basa Sunda, AI skill Bahasa Sunda.

Use this skill for source-aware Sundanese generation and review. It is not a certified translator and must keep speech level, dialect, public, legal, educational, or official text under competent speaker review when uncertain.
```

## Avoid

- stuffing irrelevant keywords,
- adding `README.md` inside every skill folder just for SEO,
- claiming "official translator",
- creating long unsourced translations to look complete,
- copying dictionary pages into the repo.
